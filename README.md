
## 📌 Introduction

This is a full-stack video sharing application where users can submit YouTube links, add descriptions, and interact with shared videos through upvotes and downvotes.

The project also includes **real-time notifications** using Socket.IO and a **background job system** using BullMQ + Redis for async processing.

### Key Features
- Authentication (NextAuth)
- Share YouTube videos via URL
- Add video descriptions
- Upvote / Downvote system
- Real-time notifications (Socket.IO)
- Background job processing (BullMQ + Redis)
- tRPC for end-to-end type safety
- Prisma ORM with PostgreSQL

---

## 🚀 Running the Application & Usage Guide

---

### ▶️ Running the Application in development

This project requires **3 services running in parallel**:

- Next.js app (frontend + API)
- Socket server (real-time notifications)
- Worker (background job processor)
- Redis (message broker)

---

#### 1. Start Redis By Docker:

```bash
docker run -d -p 6379:6379 --name redis redis:7
```

#### 2. Start Socket Server

``` bash
npm run socket
```

This starts the Socket.IO server on:

http://localhost:3001

---

#### 3. Start Background Worker

```bash
npm run worker
```

This processes queued jobs (notifications, async tasks).

---

#### 4. Start Development Server

``` bash
npm run dev
```
Access the application at:

http://localhost:3000

# 📖 Usage Guide

## 🎥 1. Share a Video

- Navigate to the /videos page, use the "Share" button
- Paste a valid YouTube URL
- (Optional) Add a description
- Submit the form

### What happens:

- Video is saved to database
- A background job is created
- Notification is triggered via Redis + Socket.IO

---

## 👍 2. Voting System

- Users can upvote or downvote videos
- Each vote is tied to a user
- Votes update in real-time on UI

---

## 🔔 3. Real-time Notifications

When a new video is shared:

- A background worker processes the event
- Redis publishes the message
- Socket server broadcasts it
- UI shows a top-left notification banner

---

## 🔐 4. Authentication

- Users must log in to:
    - Share videos
    - Vote on videos
    - Received notifications about newest shared videos
- Authentication handled via NextAuth

---

## 🧠 5. Key Features to Test (Reviewer Guide)

Make sure to verify:

- Sharing a video triggers a notification
- Notification appears without page refresh
- Votes update correctly
- Multiple users see real-time updates
- Worker is running in background terminal

---

## 🧪 Expected System Flow

User submits video  
→ tRPC API  
→ Database save  
→ BullMQ queue  
→ Worker processes job  
→ Redis publish  
→ Socket.IO broadcast  
→ Frontend notification

---

## ⚠️ Notes

- If notifications do not appear, ensure:
    - Socket server is running
    - Worker is running
    - Redis is active
- All 3 services are required for full functionality
